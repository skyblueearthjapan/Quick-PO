# 02 ─ 新規アプリ立ち上げ手順

新しい簡易ウェブアプリ（PWA 想定）を、VPS にデプロイして Cloudflare 経由で公開するまでの一気通貫手順です。

ここでは **発注書アプリ (`hachuusho`)** を例にしますが、他のアプリも同じパターンで増やせます。

---

## 設計判断（最初に決めること）

| 項目 | 推奨デフォルト | 例 |
|---|---|---|
| アプリ名 (英数) | snake or kebab | `hachuusho` |
| URL | `<app>.sakuraidenso.net` | `hachuusho.sakuraidenso.net` |
| VPS 配置先 | `/opt/<app>/` | `/opt/hachuusho/` |
| Docker compose プロジェクト名 | `name: <app>` | `name: hachuusho` |
| バックエンド | FastAPI (Python) | （他: Express / Hono など） |
| フロント | Vite + React（PWA 対応プラグイン） | |
| DB | SQLite（10〜50名規模なら十分） | |
| 認証 | Cloudflare Access（メール許可リスト） | |

## STEP 2 ─ Docker 化

### 2-1. `backend/Dockerfile`

```dockerfile
FROM python:3.12-slim
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends curl \
    && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY app ./app
ENV PYTHONUNBUFFERED=1
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=3s --start-period=10s --retries=3 \
    CMD curl -fsS http://127.0.0.1:8000/api/health || exit 1
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2-2. `frontend/Dockerfile`（多段ビルド）

```dockerfile
FROM node:22-alpine AS build
WORKDIR /app
COPY package.json ./
RUN npm install --no-audit --no-fund
COPY . .
RUN npm run build

FROM nginx:1.27-alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### 2-3. `frontend/nginx.conf`（API プロキシ込み）

```nginx
server {
  listen 80;
  root /usr/share/nginx/html;
  index index.html;
  gzip on;
  gzip_types text/plain text/css application/javascript application/json image/svg+xml;

  # PWA: service worker は no-cache
  location = /sw.js {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    try_files $uri =404;
  }

  # PWA manifest
  location = /manifest.json {
    add_header Cache-Control "public, max-age=3600";
    try_files $uri =404;
  }

  location /assets/ {
    expires 7d;
    add_header Cache-Control "public, max-age=604800, immutable";
    try_files $uri =404;
  }

  # SSE を使う場合
  location /api/events {
    proxy_pass http://api:8000/api/events;
    proxy_http_version 1.1;
    proxy_set_header Connection '';
    proxy_buffering off;
    proxy_cache off;
    chunked_transfer_encoding off;
    proxy_read_timeout 24h;
  }

  location /api/ {
    proxy_pass http://api:8000/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }

  # SPA fallback
  location / { try_files $uri /index.html; }
}
```

## STEP 3 ─ Cloudflare 設定

既存の `skyblue.earthjapan@gmail.com` の Cloudflare アカウントで作業します。

### 3-1. API トークンを発行（一時用・24時間）

`https://dash.cloudflare.com/profile/api-tokens` で **Custom Token**：

| 項目 | 設定値 |
|---|---|
| Token name | `hachuusho-setup` |
| Permissions | `Account` → `Cloudflare Tunnel` → `Edit` |
| | `Account` → `Access: Apps and Policies` → `Edit` |
| | `Account` → `Access: Organizations, Identity Providers, and Groups` → `Edit` |
| | `Zone` → `DNS` → `Edit` |
| Account Resources | Specific account → 自アカウント |
| Zone Resources | Specific zone → `sakuraidenso.net` |
| TTL | 24時間 |

トークンをコピーして環境変数に：
```powershell
$env:CF_TOKEN = "cfut_..."
```

### 3-2. Zone ID と Account ID を取得

```powershell
curl.exe -sS "https://api.cloudflare.com/client/v4/zones?name=sakuraidenso.net" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
| python -c "import sys,json;z=json.load(sys.stdin)['result'][0];print('ZONE_ID=',z['id']);print('ACCOUNT_ID=',z['account']['id'])"
```

結果を環境変数にセット：
```powershell
$env:ZONE_ID    = "7ed56a2721023a0837b78b437f1fb822"  # sakuraidenso.net の値
$env:ACCOUNT_ID = "f9e0fac29fc82b9ea283b1e9c6c655cc"
```

### 3-3. Tunnel を作成

```powershell
curl.exe -sS -X POST "https://api.cloudflare.com/client/v4/accounts/$env:ACCOUNT_ID/cfd_tunnel" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  -d '{\"name\": \"hachuusho\", \"config_src\": \"cloudflare\"}' `
| python -m json.tool
```

レスポンスから 2 つメモ：
- `result.id` → Tunnel ID（後で DNS と ingress 設定に使う）
- `result.token` → cloudflared サイドカーに渡す `TUNNEL_TOKEN`

### 3-4. DNS CNAME を追加

```powershell
$env:TUNNEL_ID = "<取得した Tunnel ID>"

curl.exe -sS -X POST "https://api.cloudflare.com/client/v4/zones/$env:ZONE_ID/dns_records" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  -d "{`\"type`\": `\"CNAME`\", `\"name`\": `\"hachuusho`\", `\"content`\": `\"$env:TUNNEL_ID.cfargotunnel.com`\", `\"proxied`\": true}"
```

→ `hachuusho.sakuraidenso.net` が Tunnel に向く。

### 3-5. Tunnel の ingress を設定

```powershell
# ingress.json をローカルで作って送る
@'
{
  "config": {
    "ingress": [
      { "hostname": "hachuusho.sakuraidenso.net", "service": "http://web:80" },
      { "service": "http_status:404" }
    ]
  }
}
'@ | Out-File -Encoding utf8 -NoNewline /tmp/ingress.json

curl.exe -sS -X PUT "https://api.cloudflare.com/client/v4/accounts/$env:ACCOUNT_ID/cfd_tunnel/$env:TUNNEL_ID/configurations" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  --data "@/tmp/ingress.json"
```

### 3-6. Cloudflare Access（認証）

許可リスト方式で社内のみアクセス可能に：

```powershell
# Group 作成（許可メール初期値）
curl.exe -sS -X POST "https://api.cloudflare.com/client/v4/accounts/$env:ACCOUNT_ID/access/groups" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  -d '{\"name\": \"Hachuusho Users\", \"include\": [{\"email\": {\"email\": \"info@sakuraidensou.com\"}}]}'
# → result.id をメモ ($GROUP_ID)

# Access App 作成（30日セッション）
curl.exe -sS -X POST "https://api.cloudflare.com/client/v4/accounts/$env:ACCOUNT_ID/access/apps" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  -d '{\"name\":\"Hachuusho\",\"domain\":\"hachuusho.sakuraidenso.net\",\"type\":\"self_hosted\",\"session_duration\":\"720h\",\"auto_redirect_to_identity\":true}'
# → result.id をメモ ($APP_ID)

# Policy 作成
curl.exe -sS -X POST "https://api.cloudflare.com/client/v4/accounts/$env:ACCOUNT_ID/access/apps/$APP_ID/policies" `
  -H "Authorization: Bearer $env:CF_TOKEN" `
  -H "Content-Type: application/json" `
  -d "{`\"name`\":`\"Allow Hachuusho Users`\",`\"decision`\":`\"allow`\",`\"include`\":[{`\"group`\":{`\"id`\":`\"$GROUP_ID`\"}}]}"
```

## STEP 4 ─ VPS にデプロイ

### 4-1. 本番用 `docker-compose.prod.yml`

```yaml
name: hachuusho

services:
  api:
    build: ./backend
    image: hachuusho-api:latest
    container_name: ho-api
    restart: unless-stopped
    volumes:
      - ./data:/data
    environment:
      SD_DB_PATH: /data/db.sqlite
    expose: ["8000"]
    networks: [ho-internal]
    healthcheck:
      test: ["CMD", "curl", "-fsS", "http://127.0.0.1:8000/api/health"]
      interval: 30s
      timeout: 3s
      retries: 3
      start_period: 10s

  web:
    build: ./frontend
    image: hachuusho-web:latest
    container_name: ho-web
    restart: unless-stopped
    depends_on:
      api: { condition: service_healthy }
    networks: [ho-internal]

  cloudflared:
    image: cloudflare/cloudflared:2025.2.1
    container_name: ho-cloudflared
    restart: unless-stopped
    depends_on: [web]
    command: tunnel --no-autoupdate run
    environment:
      TUNNEL_TOKEN: ${TUNNEL_TOKEN:?TUNNEL_TOKEN is required}
    networks: [ho-internal]

networks:
  ho-internal:
    driver: bridge
    name: hachuusho-internal
```

**ポイント**: ホストの公開ポートを一切バインドしない（Tunnel 経由のみ）。kaipoke-api 等と完全に干渉ゼロ。

### 4-2. ファイルを VPS に転送

### 4-3. `.env` を VPS に配置（秘密情報・SSH で直接書き込む）


### 4-4. ビルド & 起動

```bash

docker compose -f docker-compose.prod.yml --env-file .env up -d
```


## トラブルシュート

| 症状 | 確認ポイント |
|---|---|
| URL を開いてもログイン画面が出ない | Cloudflare DNS で CNAME が反映されているか / Tunnel が active か |
| 502 Bad Gateway | `docker logs ho-cloudflared` で `connection refused` ならコンテナ起動順を確認 |
| ログイン画面で「許可されていません」 | Access Group のメールが正しいか・Application の domain と URL が一致しているか |
| SSE が切れる | nginx.conf の `/api/events` で `proxy_buffering off` が入っているか |
| iPhone Safari で動かない | PWA manifest の `start_url` / `scope` 設定 / HTTPS 必須 |
